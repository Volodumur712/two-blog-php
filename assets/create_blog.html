Створення по черзі як в останньому уроці

Перші кроки по створенню сайта

  1. Дизайн є
  2. Спроектована база даних
  3. План реалізації
  • Встанови та налаштуй XAMPP.
  • Відредагувати файли через блокнот
  • Створи в корені папки віртуального сервера файл index.php

      • Налаштування БД

    • Зайди в phpmyadmin та створи базу даних
    • Створи таблицю (posts)
    • При створенні бази даних необхідно вказувати кодування utf8-general_ci.
    • Під час створення таблиці вказувати назви потрібно латиницею, у нижньому регістрі букв.
    • поле ID тип INT і галочку автопродовження (Чек бокс з підписом «A_I»)
    • поле назв і іншого тексту VARCHAR 255
    • структуру бази даних проекту можна намалювати на ресурсі draw.io
    • він же https://app.diagrams.net/
    • можна загуглити як робити нормальні форми баз даних
  • Завантаж файл з темплейтом собі на ПК.
  • Налаштуй віртуальний хост php-blog.test та розверни на ньому тему, яку ти завантажив.
  • Налаштуй структуру проекту та шляхи до файлів по прикладу з відео.
  • структурувати папки
  • Винеси код для посту в окремий файл post.php.
  • Розбий проект на частини(partials).
  • Для підключення до MySQL із PHP нам треба вказати налаштування підключення: адресу сервера, логін, пароль, назву бази даних
  • додати логіку навігації по сторінках і частинах
  • Створи таблицю для категорій в своїй базі даних elen_blog.
  • Встанови програму Navicat для управління базою даних. (Можеш використовувати будь-які інші схожі клієнти).
  • Налаштуй вивід постів з бази даних на сторінку post.php.
  • Додай сторінку авторизації користувача login.php.
  • Налаштуй механізм авторизації юзерів.
  • Додай до проекту файли like.php, api.php, api.js та директорію api.
  • Додай маршрут додавання лайку та протестуй його по прикладу з відео.
  • Додай функціонал додавання лайку як це показано на відео.
  • Додай функціонал динамічного виводу кількості лайків до посту по прикладу.
  • Додай до свого проекту можливість хешувати паролі користувачів при реєстрації та авторизації для безпечного зберігання паролів.
  • При успішній авторизації встанови “кукі” по прикладу з відео.
  • Обери будь який безкоштовний хостинг та розверни на ньому свій сайт.
  • Для цього перенеси файли проекту та базу даних, налаштуй з’єднання, DNS(на безкоштовних хостинг провайдерах зазвичай вже налаштовано)
  • та попрактикуйся в підключенні до серверу користуючись одним із протоколів для передачі даних(по прикладу з відео).
